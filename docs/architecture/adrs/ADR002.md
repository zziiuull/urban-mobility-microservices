# ADR 002 - NGINX como API Gateway e Load Balancer

## Título

Elemento adicional: NGINX como API Gateway e balanceador de carga

## Situação

Aceita

## Contexto

O sistema é composto por múltiplos microsserviços, que precisam ser expostos de forma unificada para clientes externos.

É necessário um único ponto de entrada para a API, capaz de:
- balancear carga entre várias instâncias de um mesmo serviço;

## Decisão

Utilizar NGINX como API Gateway e load balancer

- Clientes acessam um único endpoint
- NGINX executa balanceamento de carga entre múltiplas instâncias de um mesmo serviço

## Consequências

Positivas:

- Unificação de acesso à API

Negativas:

- NGINX passa a ser ponto único de falha
