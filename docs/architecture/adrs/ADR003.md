# ADR 003 – Uso de Redis como Cache Distribuído

## Título

Elemento adicional: Redis como cache distribuído para leitura intensiva

## Situação

Aceita

## Contexto

Algumas operações apresentam alta taxa de leitura e exigem baixa latência, como:
leitura de preços dinâmicos por zona de cidade;


## Decisão

Adotar Redis como cache distribuído, com uso específico por serviço:

- Pricing Service: armazenar preços e tarifa dinâmica

Cada microsserviço continua tendo seu próprio Postgres como fonte de verdade; Redis será usado apenas como camada de leitura rápida.

## Consequências

Positivas

- Redução significativa da latência em consultas de preço.

Negativas

- Maior complexidade operacional, como: preenchimento e invalidação de cache.
- Redis torna-se um componente adicional a ser monitorado.
